# Explore With Variants

## Overview

Some tasks are not about finding the answer quickly. They are about searching the space well.

## When to Use

- Design and UX exploration
- Naming, writing, layout, or concept work with multiple plausible directions
- Early-stage product or interface thinking
- Situations where the user wants options, tradeoffs, or mix-and-match decisions

## Rules

- Default to multiple directions when the space is genuinely open.
- Make the differences meaningful: structure, style, interaction, framing, or risk profile.
- Prefer one primary artifact with switchable or comparable variants over many disconnected forks.
- Do not force false variety when the task is actually deterministic.

## Process

1. Decide whether the task is exploratory or deterministic.
2. If exploratory, choose 2-4 useful axes of variation.
3. Produce clearly distinct options, not cosmetic rewrites.
4. Explain what each option optimizes for.
5. Converge only after the options are visible and comparable.

## Common Mistakes

- Giving one polished answer when the user really needs search
- Producing fake variants that differ only in wording
- Creating many file forks when one main artifact could hold the variants
